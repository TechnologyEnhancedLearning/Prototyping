This addresses three main points:

  • Removing the boarder from __key, __Value and action (DTs and DDs) and setting on the row means the border maintains a consistent width regardless of every item needing an action.
  • This implementation allows the use of multiple values (DDs).
  • Setting a fixed width on __keys and __actions so items are a consistent length across multiple Summary lists (DLs) on a page.
